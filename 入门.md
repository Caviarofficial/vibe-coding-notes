# 切换模式
1. 按下**shift+tab**可切换，三种模式：
    1. ask before edits:干什么修改之前都会问
    2. edit automatically:自动修改代码
    3. plan：专门做计划用的
    4. Yolo模式：赋予更高级别的授权，包括创建文件夹、删除文件等，需要通过命令
```
claude --dangerously-skip-permissons
```
方可进入

# 关于执行命令
直接在对话中输入命令是不行的，它会认为是一条指令，所以只能够用”**请帮我执行指令XXXXXXXXXXXXX**“的方式去进行执行

# 关于claude。md
这个文件发挥的作用就是**全局提示词集合**，存储一些全集注意的规范，每次大的对话中都建议提一嘴**参考我的claude。md**文档，里面存放包括但不限于：
1. 常用 bash 命令
2. 核心文件和实用函数
3. 代码风格指南
4. 测试说明
5. 代码库规范（例如，分支命名、合并与变基等）
6. 开发环境设置（例如，pyenv 使用、哪些编译器有效）
7. 项目特有的任何意外行为或警告
8. 您希望 Claude 记住的其他信息
例子：
```
    # Bash命令
  - npm run build: 构建项目
  - npm run typecheck: 运行类型检查器

  # 代码风格
  - 使用ES模块（import/export）语法，而不是CommonJS（require）
  - 尽可能使用解构导入（例如 import { foo } from 'bar'）
  - 代码加上中文注释


  # 工作流程
 - 在完成一系列代码更改后务必进行类型检查
 - 为了性能考虑，优先运行单个测试，而不是整个测试套件
```

# 关于会话管理
1. 暂停：**按一次esc**，继续执行直接跟他说**继续执行**就行了
2. 回滚到历史对话节点：**按两次esc**,h或者对代码不满意直接说**回滚到上次代码**
3. 手动延长对话续命：**compact**
> 对话提示“context left until auto-compact:3%"的时候说明长度不够了
4. 翻上一条或者上几条打出去的句子：**按↑即可**
5. （**重点**） 退出claude后，恢复到上次对话：
   1. 打开claude的时候不是输入claude，而是输入**claude -c**：恢复到关闭前最后一条对话
   2. 同理**claude -r**：可以在关闭前的20条对话任选一条恢复
   > 在退出claude的时候，不能直接关闭，最好使用命令**exit**先退出到终端

# 关于资源监控
在运行claude前，在power shell中安装npx包，可以实时监控token用量：
1. 按天查看：**npx ccusage@latest**
2. 实时监控：**npx ccusage blocks --l**

# 批量任务高效处理：
如果需要执行几十个重复任务（比如批量生成文章章节），用脚本式用法：
1. 把任务**按行写入TASK.md**,必须一行一个任务
2. 运行命令：
```
cat TASK.md | while IFS= read -r line; do echo $line; claude -p "$line" --debug; done
```

# 防备幻觉与虚假成果
1. 在claude.md中加入**每次宣称成果必须附上证据链接**
2. 定期反问**真的完成了吗？**、**有完成证据吗？**
